# Kontingent für eine Organisation festlegen
{: #determinequota}

Wenn Sie eine Organisation in {{site.data.keyword.Bluemix_notm}} erstellen, geben Sie Infrastrukturressourcen an, die Ressourcen wie Hauptspeicher, maximale Anzahl an Serviceinstanzen und maximale Anzahl an Routen umfassen. Das Kontingent, das einer Organisation zugeordnet wird, stellt die Ressourcen dar, die innerhalb der Organisation verfügbar sind. Sie verwalten das Kontingent selbst und entscheiden, wie die Ressourcen auf die Organisation verteilt werden. Sie legen das Kontingent für die Organisation fest, wenn Sie die Organisation erstellen. Sie können dieses Kontingent auch jederzeit auf der Übersichtsseite der Organisation ändern. Die tatsächliche Kapazität der Organisation wird durch die gesamte physische Kapazität der Umgebung gebunden, die sich aus der Anzahl der Diego-Zellen ergibt, von denen diese Umgebung optimiert wird.

Bei gemeinsamer Verwendung (Shared) und dedizierter Verwendung (Dedicated) können Sie die Anzahl der Diego-Zellen für die Umgebung erhöhen. Sie sind für die Bereitstellung aller Ressourcen verantwortlich, die für den Betrieb Ihres Unternehmens in dieser Umgebung erforderlich sind.
